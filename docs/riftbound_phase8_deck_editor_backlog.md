# Riftbound Simulator — Phase 8 Backlog (Deck Editor)
_Last Updated: 2026-03-06_

## Epic
**Deck Editor Interface and Validation**

Provide tools to build legal decks once the game engine is stable.

---

## Feature 8.1 — Deck Builder UI
Allow players to construct decks.

### User Stories
- **US8.1.1**: As a player, I must browse available cards.
- **US8.1.2**: As a player, I must add cards to my deck.
- **US8.1.3**: As a player, I must remove cards from my deck.

---

## Feature 8.2 — Deck Validation
Ensure decks follow game rules.

### User Stories
- **US8.2.1**: As the system, I must enforce minimum deck size.
- **US8.2.2**: As the system, I must enforce copy limits.
- **US8.2.3**: As the system, I must validate rune deck requirements.

---

## Feature 8.3 — Deck Export
Allow decks to be used in matches.

### User Stories
- **US8.3.1**: As a player, I must export my deck to a decklist format.
- **US8.3.2**: As the system, I must load decks into matches.
- **US8.3.3**: As the engine, I must ensure decklists remain immutable during matches.
